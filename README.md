# 原稿トラッカー

漫画・イラスト原稿の進捗をブラウザ上で管理するツールです。  
**インストール不要。URLを開くだけで使えます。**

🔗 **[ツールを開く](https://kinpiras155.github.io/github.io/)**  


---

## 特徴

- 書き出し済みの PNG / JPG をフォルダごとドラッグ＆ドロップで読み込み、サムネイル一覧表示
- ページごとにステータス・メモ・修正チェックリスト・作業時間などを記録
- 全ページの修正項目を「修正リスト」サイドバーで一覧管理
- 見開き表示（右綴じ・左綴じ対応）
- カラーテーマ 5種から選択
- **外部送信ゼロ。画像もメモも一切サーバーへ送られません**

---

## クイックスタート

1. 上記URLをブラウザで開く
2. 「📁 フォルダを読み込み」で原稿フォルダを選択、またはウィンドウへドラッグ＆ドロップ
3. サムネをダブルクリックすると詳細パネルが開きます

## 基本的な使い方

### 画像の読み込み

| 操作 | 説明 |
|---|---|
| 📁 フォルダを読み込み | フォルダを丸ごと一括読み込み |
| ＋ 画像を追加 | 個別に画像を追加（既存ページより後ろに追加） |
| ウィンドウへD&D | フォルダ・ファイルをドラッグ＆ドロップ |
| カードへD&D | そのページの画像を差し替え（メタデータは引き継ぎ） |

- **初回読み込み時**はファイル名に含まれる最初の数字をページ番号として自動認識します。  例）ex_p001.png→p.01として認識
- **2回目以降の追加**では、既存の最大ページ番号の後ろに順番に追加されます。

### 詳細パネル（ダブルクリックで開く）

| 入力項目 | 内容 |
|---|---|
| ページ番号 | ページ順の管理に使用。並び替え後は自動で振り直されます |
| 章 / 話 | 任意テキスト（例：第3話、Ch.2） |
| 作業時間 | 時間・分で入力。全ページ合計がヘッダーに表示されます |
| ステータス | 未着手 / ネーム / 下描き / ペン入れ / 仕上げ / 完了 / 要修正 |
| タグ | プリセット7種＋自由入力。複数付与可 |
| 修正チェックリスト | 修正箇所を1行ずつ管理。完了チェックで修正リストサイドバーに反映 |
| メモ | 自由テキスト（演出意図・保留事項など） |

### ステータスの意味（推奨運用）

| ステータス | 用途 |
|---|---|
| 未着手 | まだ何も手をつけていないページ |
| ネーム | ネーム・構成段階 |
| 下描き | 下描き完了 |
| ペン入れ | ペン入れ完了 |
| 仕上げ | トーン・仕上げ作業中 |
| 完了 | 書き出し・入稿まで完了 |
| 要修正 | 修正が必要と判断したページ |

### ページの並び替え

グリッド表示・ページ番号順・絞り込みなし・検索なしのとき、カードをドラッグして並べ替えができます。  
確定後にページ番号が1から自動で振り直されます。

**複数ページを束で移動する**には、Shift＋クリックで複数選択してから、選択中のカードをドラッグしてドロップします。

### 複数選択と一括操作

1. カードをクリックして1件選択（再クリックで解除）
2. **Shift＋別のカードをクリック**で範囲選択
3. 画面下部のバーからステータス一括変更・タグ一括付与・一括削除が可能

ESCキーで選択を解除できます。

### 修正リストサイドバー

「📋 修正リスト」ボタンで表示。全ページの修正チェックリストをページ順に一覧表示します。

- **未完了**と**完了済み**でセクションを分割（完了済みは折りたたみ）
- 各行をクリックするとそのページの詳細パネルが開く
- チェックボックスをその場でオン/オフ可能

### 見開き表示

フィルター行の「見開き」ボタンで切り替えます。右綴じ読み順（右から左）で見開きを表示します。

- **1ページ目から見開き**（1-2, 3-4…）：ページ数が奇数始まりの場合
- **1ページ目は単独**（1 / 2-3, 4-5…）：表紙など1ページ目が単独の場合

### ジャンプ機能

フィルター行のページ番号入力欄に番号を入れてEnterまたは「↪ ジャンプ」で、そのページへスクロールします。100p超の原稿でも一発でジャンプできます。

## メタデータの保存について

### 保存先と保存内容

ステータス・メモ・修正チェックリスト・ページ番号・作業時間などの**メタデータはブラウザの localStorage に保存**されます。  
**画像ファイルそのものは保存されません。外部サーバーにも一切送信されません。**  
ツールを開くたびに同じフォルダを読み込み直してください。

### ⚠️ GitHub Pages 特有の注意点

**メタデータはこのURLにアクセスしているブラウザに保存されます。**

- 同じURLを別のブラウザ・別のPCで開いても、記録は共有されません
- ブラウザの「閲覧データを消去」「キャッシュ削除」などを行うと記録が消える場合があります
- **バックアップJSONの定期書き出しを強くおすすめします**

### メタデータのキーはファイル名です

```
p001.png → { status: "ink", memo: "...", checklist: [...] }
p002.png → { status: "fix", checklist: [...] }
```

**ファイル名を変えると記録との紐付けが切れます。** クリスタ等で上書き書き出しする場合はファイル名を変えないようにしてください。

### バックアップ・復元

| 操作 | 内容 |
|---|---|
| ⤓ 書き出し（バックアップ） | メタデータをJSONファイルとして書き出し |
| ⤒ 読み込み（復元） | バックアップJSONを読み込んで復元 |

復元後に画像フォルダを読み込み直すと、バックアップ時点のステータス・メモが復元されます。

### 別のPCや別ブラウザへの移行手順

1. 移行前の環境で「書き出し（バックアップ）」→ JSONファイルを保存
2. 移行先のブラウザで同じURLを開く
3. 「読み込み（復元）」でJSONを読み込む
4. 画像フォルダを読み込み直す

## 設定（⚙ 設定ボタン）

| 設定項目 | 内容 |
|---|---|
| カラーテーマ | 墨（ダーク）/ 藍（ダーク）/ 黒檀（ダーク）/ 紙（ライト）/ 霧（ライト） |
| 削除前に確認ダイアログを表示する | オフにすると削除確認をスキップ。削除ダイアログの「次回から確認しない」と連動 |
| メタデータを全消去 | 記録をすべて消去して初期状態に戻す（画像は消えません・元に戻せません） |

## よくある質問

**Q. ブラウザの履歴を消去したら記録が消えた**  
A. localStorageはブラウザの「サイトデータ」に含まれるため、消去対象になる場合があります。バックアップJSONから復元してください。

**Q. 別のPCで開いたら記録がない**  
A. メタデータはブラウザごとに独立しています。バックアップJSONを移行先で読み込んでください。

**Q. フォルダを読み込み直したら順番がおかしい**  
A. ページ番号が保存されていない場合、ファイル名の数字を読んで並べます。一度詳細パネルでページ番号を設定するか、ドラッグで並び替えると以降は保存されます。

**Q. 複数人で進捗を共有したい**  
A. バックアップJSONをファイルで共有し、受け取った側が「読み込み（復元）」で反映する運用が使えます。リアルタイム同期には対応していません。

**Q. データはサーバーへ送られますか？**  
A. 送られません。このツールは外部への通信をまったく行いません。原稿画像・メモ・ステータスのすべてがブラウザ内で完結します。GitHub Pagesはファイルの配信のみ行い、利用者のデータには関与しません。

**Q. オフラインでも使えますか？**  
A. 初回はURLにアクセスしてページを読み込む必要がありますが、一度開いた状態ではオフラインでも操作できます（ブラウザキャッシュに依存します）。
